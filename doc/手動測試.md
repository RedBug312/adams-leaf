調整 flows.json 中 TT 資料流的大小，可以有效地排擠網路資源，使 AVB 資料流改走別條路徑。例如

```js
{
    "src": 4,
    "dst": 2,
    "size": 2000, // 若調大這個值至20000，則4->2中間幾乎變成斷路
    "period": 10,
    "max_delay": 100,
    "offset": 20
}
```

目前測資下最好的結果：
```
[4, 2]
[0, 4, 3, 2]
0.285
[0, 4, 3, 2]
0.285
[0, 1, 3, 2]
0.28
[0, 1, 3, 2]
0.28
[0, 4, 2]
0.325
sum = 1.4549999999999998
```
